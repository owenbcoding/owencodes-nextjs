Rebuilding my portfolio from scratch: what I did differently, moving from a Laravel stack to a Next.js stack.

**Highlights:**

- Next.js (App Router), React, and TypeScript for a simple, fast stack.

- File-based blog posts (MDX) so the site can run without a database for content.

- Clean, low-cost deployment on Vercel with minimal configuration.

- The main reason for the switch: I couldn't keep affording Laravel Cloud's hosting costs for a personal portfolio, so I moved to a JavaScript framework (Next.js) where the free / cheaper tiers made a lot more sense.

---

This dev log covers the decisions and changes I made when rebuilding this portfolio: stack choice, design approach, and how I structured the project for maintainability and deployment (including file-based posts and moving off Laravel Cloud).

The first time I made my portfolio, it was okay—but I didn't like the UI or the UX. My CSS skills weren't the best, and frontend wasn't something I enjoyed, so I spent a lot of free time improving it into what you see now. Originally it was built on a Laravel stack and deployed to Laravel Cloud, but the hosting cost for a personal site I wasn't monetizing started adding up fast. Rather than keep paying to keep a portfolio online, I decided to rebuild it on a JavaScript framework—Next.js—which I can host cheaply (or free) on platforms like Vercel while still getting SSR, great DX, and a modern frontend stack.
